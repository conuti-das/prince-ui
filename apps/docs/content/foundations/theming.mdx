# Theming

prince-ui kennt drei Themes, gesteuert über das Attribut `data-theme` am `<html>`-Element:

- **Dunkel** (`data-theme="dark"`) — der Default und Fallback. Dunkle System-Farben, grüner Akzent.
- **Hell** (`data-theme="light"`) — die helle Variante mit WCAG-AA-justiertem Sekundärtext.
- **Cu** (`data-theme="cu"`) — das CONUTI-CI-Theme: CI-Grün `#a0d22b`, Inter-Font, grünes Bento-Mesh.

Ohne `data-theme` folgt prince-ui dem Betriebssystem (`prefers-color-scheme`) und fällt bei
fehlender Präferenz auf **Dunkel** zurück. Hell und Dunkel sind die neutralen System-Themes,
Cu ist die CONUTI-CI-Variante.

## setTheme / getTheme

Das Umschalten erledigt die exportierte Funktion `setTheme` aus `@conuti-das/prince-ui`. Sie
setzt bzw. entfernt `data-theme` am Wurzelelement. `null` bedeutet „dem System folgen":

```ts
import { setTheme, getTheme, type PrinceTheme } from '@conuti-das/prince-ui';

setTheme('cu');     // erzwingt das Cu-Theme (CONUTI-CI)
setTheme('light');  // erzwingt Hell
setTheme(null);     // wieder dem OS folgen
getTheme();         // => "cu" | "light" | "dark" | null
```

Probiere es direkt aus — die Buttons schalten das Theme dieser Doku-Seite live um:

<Example code={`<div style={{ display: 'flex', gap: 12, flexWrap: 'wrap' }}>
  <Button variant="filled" onPress={() => setTheme('dark')}>Dunkel</Button>
  <Button variant="tinted" onPress={() => setTheme('light')}>Hell</Button>
  <Button variant="tinted" onPress={() => setTheme('cu')}>Cu</Button>
  <Button variant="plain" onPress={() => setTheme(null)}>System folgen</Button>
</div>`} />

## So wirkt der Wechsel

Weil alle Komponenten ausschließlich über `--prn-*`-Tokens stylen, genügt der Attribut-Wechsel
am `<html>` — der gesamte Token-Satz wird neu belegt und jede Fläche, Schrift und Linie zieht
ohne weiteres Zutun mit. Es gibt kein per-Komponente-Theming und keine Theme-Props.

<Example code={`<div style={{
  padding: 20,
  background: 'var(--prn-bg-elevated)',
  border: 'var(--prn-card-border)',
  borderRadius: 'var(--prn-radius-card)',
  boxShadow: 'var(--prn-shadow-raised)',
  color: 'var(--prn-label)'
}}>
  <div style={{ font: 'var(--prn-text-headline)' }}>Diese Karte folgt dem Theme</div>
  <div style={{ font: 'var(--prn-text-subhead)', color: 'var(--prn-label-2)', marginTop: 4 }}>
    Schalte oben um — Fläche, Akzent und Text wechseln mit.
  </div>
  <div style={{ marginTop: 12, color: 'var(--prn-accent)', font: 'var(--prn-text-headline)' }}>
    Akzentfarbe (--prn-accent)
  </div>
</div>`} />

## Tokens-CSS laden

Damit die Tokens überhaupt existieren, muss das Tokens-CSS einmal am App-Einstieg importiert
werden — gefolgt vom Komponenten-CSS von prince-ui:

```ts
import '@conuti-das/prince-ui-tokens/tokens.css';
import '@conuti-das/prince-ui/styles.css';
```

<DoDont kind="do">Theme über `setTheme(...)` bzw. `data-theme` am `<html>` zentral steuern — alles zieht automatisch mit.</DoDont>
<DoDont kind="dont">Einzelne Komponenten mit eigenen Farb-Props „theme-en" oder zwei Themes gleichzeitig auf verschachtelten Elementen erzwingen.</DoDont>

## Accessibility

Die drei Themes sind nicht nur kosmetisch: Hell justiert `--prn-label-2` auf 75 %
Deckkraft, damit Sekundärtext WCAG-AA erreicht, und jedes Theme setzt das passende
`color-scheme` (`dark`/`light`), sodass native Controls und Scrollbars korrekt rendern. Wenn
du einen Theme-Umschalter ins UI baust, gib ihm ein zugängliches Label (`aria-label="Theme"`)
und respektiere die System-Präferenz als sinnvollen Default, indem du initial `setTheme(null)`
verwendest.
